# Spring Sunshine Precourse - 날씨 조회 서비스

## 구현할 기능 목록
- 사용자가 도시 이름을 입력하여 날씨를 조회할 수 있는 REST API 제공
- 도시별 위도·경도 매핑을 DB에 적재하고 조회하는 컴포넌트 구성(서울, 도쿄, 뉴욕, 파리, 런던, 시드니 등 최소 5개 도시 지원)
- 날씨 코드와 설명 매핑을 DB 테이블로 관리하고 조회하는 컴포넌트 구성
- Open-Meteo API를 호출하는 외부 연동 어댑터 구현
- 현재 기온, 체감 온도, 하늘 상태, 습도 정보를 담은 응답 DTO 생성
- 조회한 데이터를 기반으로 한 줄 요약 문장을 생성하는 로직 구현
- 입력 검증, 공통 응답 구조, 에러 처리 및 테스트 코드 작성
- Kotlin 기반으로 JPA·Flyway·Thymeleaf·Validation 등 Gradle 의존성을 모두 활용

## 구현 전략
- 계층 구조는 `api`(컨트롤러/DTO) → `application`(파사드) → `component`(소형 컴포넌트) → `infrastructure`(외부 어댑터) 순으로 분리한다.
- 요청 DTO에는 Bean Validation을 적용하고, 실패 시 공통 응답 포맷을 사용한다.
- 도시 좌표는 Flyway로 초기화된 H2 DB에 저장하고, JPA 리포지토리를 통해 조회한다.
- 날씨 코드-설명 매핑도 DB 테이블과 JPA 리포지토리로 관리해 외부 API의 코드 값을 설명으로 변환한다.
- 외부 API 호출은 `RestTemplate`를 사용하는 어댑터로 분리하여 타임아웃 및 오류 매핑을 처리한다.
- 파사드는 좌표 조회 → 외부 API 호출 → DTO 매핑 → 요약 문장 생성을 순차적으로 오케스트레이션한다.
- 모든 메서드는 하나의 역할만 수행하도록 분리하고, `else`/`switch`/3항 연산자를 사용하지 않는다. 들여쓰기는 4 spaces, 깊이는 2단계까지만 유지한다.
- 테스트는 JUnit5와 AssertJ를 사용해 좌표 매핑, 요약 문장 생성, 파사드 흐름, 컨트롤러 API 응답을 검증한다.
